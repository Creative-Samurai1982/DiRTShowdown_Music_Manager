CHANGELOG

DiRT Showdown Music Manager

All notable changes to this project will be documented in this file.

The format is loosely based on Keep a Changelog, adapted for this project.

[0.0.3] – Stable Release Candidate Phase
Added

Automatic cleanup of vgmstream sidecar files (*.dic#*.wav)

Version-aware interactive build system

Versioned EXE naming (e.g. ShowdownMusicManager_v0.0.3.exe)

Versioned release folder generation

UK English manual (PDF)

Structured first-run workflow documentation

Clear explanation of DIC/WIM architecture in manual

Author contact details for feedback and feature requests

Improved

WAV cleanup logic now safely targets DIC sidecar files only

Backup-first safety model clarified in documentation

Rich CLI workflow simplified for first-time users

Manual rewritten to be workflow-based instead of feature-based

UK spelling standardised (Licence, categorisation, behaviour, etc.)

Fixed

Sidecar WAV files not being deleted automatically

Documentation not appearing correctly in packaged builds

Inconsistent tool resolution for vgmstream CLI

Minor spelling inconsistencies

Duplicate loop in About menu

Path resolution issues in packaged EXE

Notes

This version represents a stable injection pipeline.

Profiles, injection, cleanup, and restore features verified working.

Considered baseline for future enhancements.

Versioning Philosophy

This project currently uses a 0.x version scheme during stabilisation.

0.0.x → Structural stabilisation & bug fixes

0.1.x → Feature-complete pre-release

1.0.0 → First fully stable public release
